# v5.5.2 发布教训

## 坑点 33: 生态包子包文件缺失

**症状：** canon/guard/mnemonic 子目录各只有 SKILL.md 一个文件。

**漏了什么：**
- README.md — 子包各自的 README
- CHANGELOG.md — 子包各自的更新日志
- LICENSE — MIT 许可证
- references/*.md — 子包各自的参考文档（canon 5个、guard 3个）
- scripts/*.py — 子包脚本（canon 有 2 个）

**根因：** 打包脚本只 `cp $BASE/<pkg>/SKILL.md`，没有拷贝其余文件。

**修复：** 逐一核对源目录与桌面包，每个子包保证与 `~/.hermes/skills/software-development/<pkg>/` 内容一致。

---

## 坑点 34: 子包 README 标题版本号滞后

**症状：** guard README 标题仍写 `v4.8.1`，canon 仍写 `v2.7.1`，mnemonic 仍写 `v3.5.2`。

**根因：** 版本升级只改了 SKILL.md 的 frontmatter `version:` 和激活消息，忘了 README.md 标题和描述行中也硬编码了版本号。

**修复：** 发布前 grep 版本号时必须覆盖所有子包的 README.md。子包 README 的 H1 标题和 description blockquote 都含版本号。

---

## 坑点 35: 生态包 README 太敷衍（第二次）

**v5.5.1 教训：** 生态包 README 写了 100 行，单包 README 有 304 行。用户批「太敷衍」→ 补到 304 行。

**v5.5.2 再犯：** 又只写了 100 行。用户批「太敷衍，没吸取教训」→ 补到 338 行。

**判定：** 大合集 README 行数不得低于单包 README。架构图 + 所有组件详解 + 安装配置 + 命令表 + 版本历史 + FAQ，一个不能少。
